Features

Everything you need, nothing you don't

A carefully crafted set of features that covers real-world data table needs without the bloat.

Zero Dependencies

No jQuery, no frameworks, no bloat. Just ~5KB of pure JavaScript that works everywhere out of the box.

Reactive Data

Powered by JavaScript Proxy. Mutate your data and the table updates automatically — no manual re-renders needed.

Built-in Search

Instant full-text search with configurable debounce. Filters across all columns without any extra setup.

Column Sorting

Click-to-sort on any column, ascending or descending. Supports custom sort fields for complex data types.

Pagination

Built-in pagination with configurable page size. Navigate with nextPage() / prevPage() and track state easily.

Plugin System

Transform cell data with plugins. Format dates, currency, badges — anything — with a simple transform function.

Quick Start

Up and running in minutes

Install via npm or load from a CDN, then create your first reactive table.

npm CDN
bash 
# Install with npm
npm install ezytables

# Or load directly from a CDN
# <script type="module" src="https://esm.sh/ezytables"></script>
javascript 
import { EzyTables } from 'ezytables';

const data = [
  { name: 'Alice Johnson',  role: 'Engineer',  status: 'Active'  },
  { name: 'Bob Smith',      role: 'Designer',  status: 'Active'  },
  { name: 'Carol Davis',    role: 'Manager',   status: 'Away'    },
  { name: 'Dan Wilson',     role: 'Engineer',  status: 'Active'  },
];

const table = new EzyTables({
  data,
  clientEnabled: true,
  client: { limit: 100, perPage: 10 },
  columns: [
    { name: 'name',   label: 'Name'   },
    { name: 'role',   label: 'Role'   },
    { name: 'status', label: 'Status' },
  ],
  target: '#my-table',
});

// Search with debounce
searchInput.addEventListener('input', (e) => {
  table.setSearchDebounced(e.target.value);
});

// Sort by column
table.sortData('name', 'asc');

// Pagination
table.nextPage();
console.log(table.getShowingInfo()); // "Showing 1 to 10 of 50 items"
Usage Modes

Two ways to render your tables

Use a declarative target selector or bring your own render function for full control.

🎯 Target Table Mode

Point EzyTables at a DOM element and it builds the entire table for you — header, body, pagination, search, and all.

javascript 
const table = new EzyTables({
  data: myData,
  clientEnabled: true,
  client: { limit: 100, perPage: 5 },
  target: '#my-table',
  columns: [
    { name: 'name',  label: 'Name'  },
    { name: 'email', label: 'Email' },
  ],
});

// That's it! EzyTables handles
// the full table UI for you.

🎨 Render Function Mode

Take full control of rendering. EzyTables manages the data pipeline; you decide how it looks on screen.

javascript 
const table = new EzyTables({
  data: myData,
  clientEnabled: true,
  client: { limit: 100, perPage: 5 },
  renderFunction: (rows) => {
    const tbody = document
      .querySelector('#tb tbody');
    tbody.innerHTML = rows.map(r =>
      `<tr><td>${r.name}</td>
       <td>${r.email}</td></tr>`
    ).join('');
  },
});
Plugin System

Extend with plugins

Transform any cell data with a simple plugin interface. Format dates, render badges, sanitize HTML — your call.

javascript 
// Define a plugin that transforms specific fields
const statusBadge = {
  name: 'statusBadge',
  field: 'status',   // or an array: ['status', 'priority']
  transform(value, element, parent) {
    const color = value === 'Active' ? '#10B981' : '#F59E0B';
    return `<span style="color:${color}">● ${value}</span>`;
  }
};

const dateFormat = {
  name: 'dateFormat',
  field: 'date',
  transform(value) {
    return new Date(value).toLocaleDateString('en-US');
  }
};

// Register during initialization
const table = new EzyTables({
  data, target: '#my-table',
  plugins: [statusBadge, dateFormat],
  // ...other options
});

// Or register later at runtime
table.registerPlugin(statusBadge);

API Reference

Simple, predictable API

A handful of methods is all you need. Every method does one thing well.

Method Description Returns
constructor(opts) Initialize a new EzyTables instance with EasyTablesOptions. EzyTables
setSearchDebounced(query) Set the search query with built-in debounce. Filters across all columns. void
sortData(column, order) Sort table data by a column name. Order is "asc" or "desc". void
nextPage() Navigate to the next page and re-render the table. void
prevPage() Navigate to the previous page and re-render the table. void
getCurrentPage() Get the current page number (1-based). number
getTotalPages() Get the total number of pages based on data size and per-page setting. number
getShowingInfo() Get a human-readable info string, e.g. "Showing 1 to 10 of 50 items". string
setPerPage(perPage) Change the number of items displayed per page and reset to page 1. void
registerPlugin(plugin) Register a data transform plugin at runtime. Plugin applies to matched fields. void

Constructor Options EasyTablesOptions

TypeScript 
interface EasyTablesOptions {
  clientEnabled?: boolean;        // Enable client-side mode (default: true)
  data?: any[];                   // Data source array (client-side only)
  server?: {
    api_url: string;              // API endpoint for server-side fetching
    headers?: Record<string, string>; // Optional request headers
    limit: number;                // Items per server request
    page: number;                 // Starting page number
    dataNames: string;            // Dot-path to data in response
  };
  client?: {
    limit: number;                // Max items to process
    perPage?: number;             // Items per page (default: 10)
  };
  renderFunction?: (data: any[]) => void; // Custom render callback
  targetn>?: string;                // CSS selector for auto-rendered table
  columns?: Column[];              // Column definitions
  rows?: any[];                    // Pre-defined row data
  classes?: Classes;               // Custom CSS class overrides
  plugins?: Plugin[];              // Data transform plugins
  hideDetails?: {
    header?: boolean;             // Hide search/per-page header
    footer?: boolean;             // Hide pagination footer
  };
}
